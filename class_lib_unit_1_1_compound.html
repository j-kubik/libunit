Class representing Compound unit or dimension.

Compound types are types consisting of more than one simple type. Compound class contains a list of types as template arguments. This list represents list of units or dimensions.

Note that a type is a compound type if it contains more than one simple type; ie.

// T and U are simple types
Power<T, 2> // Not a compound type
Compound<Power<T,2>, U> // A compound type
Power<Compound<T,U>,3> // A compound type
Power<Compound<T>, 2> // Not a compound type
Power<Power<Compound<Power<T,2>>,2,3> // Not a compound type
Power<Power<Compound<Power<T,2>, U>,2,3> // A compound type

Distinction between capital-C Compound type (type based on Compound class template), and compound type (type that contains more than one simple type) is important in this documentation.
